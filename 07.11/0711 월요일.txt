1교시 : 부트스트랩
	BootStrap : 트위터에서 만든 프론트엔드 프레임워크. 
		얘네들이 만든 클래스만 부여하면 미리 만들어놓은 디자인과 기능이 작동됨.
		여러 프론트엔드 프레임워크가 있지만 제일 기본이 되고 하나만 알면 다른건 자동으로 쓸수있다.
		2021년 5월에 v5가 나왔고 jquery에 대한 의존성이 사라졌다.
	<사용하는법>
	1. Bootstrap 소스 코드 다운받기(내가쓸려는 html에서 쓸수있도록)
	2. 어떤 디자인을 적용할지 검색
	3. 검색결과 복붙
	4. 커스터마이징(적용결과가 마음에 안들면 다른걸로)
	
	CSS적용방법 3가지.   태그안에 인라인으로 넣든가,   Style태그에 별도로 빼든가, link 외부파일로 참조하든가
	CSS문법 : 선택자{ 키 : 밸류 }
		예) H1{
			   color : red;
			}
			
	천재개발자들이 이미 인생을 갈아넣은 css를 무료로 오픈소스화 했다.
	그걸 안쓸 이유가 없다. 디자인 하느라 고생하지말고 이미 구현된것을 갖다쓸수있는 능력을 갖춰야한다.
	백엔드 기능 마찬가지.
	부트스트랩으로 개발하면 자동으로 반응형 디자인이 된다. -> 기기의 해상도에 맞게끔 알아서 디자인이 변경되는것
	
	이력서에 보유기술에 css프레임워크(bootstrap, tailwind)  정도 하면 좋음
	
	클래스만 바꿨는데 왜 디자인이 적용되나요? -> 우리가 1번에 받은 css파일에는 .해당클래스{} 에 해당하는 소스가 있기 때문
	
	면접질문.
	화면 디자인이 좋은데 어떻게 했나요?
		-> 우리팀원중에 프론트엔드 쪽으로 취업하려고 하는애가 있어서, 실무경험이 있는애가 있어서
		직접 디자인안하고 우리가 사용한 기술을 고객에게 잘 표현할수있는 좋은 디자인을 찾아서 적용하는데 집중했습니다.
		왜냐하면 디자인을 직접하는일은 제 취업 방향에 맞지않고 더 잘할수도 없다고 생각했기 때문입니다.
	화면 디자인이 나쁜데 어떻게 했나요?
		-> 화면디자인 맡은팀원이 책임감이 없어서 그렇게 되었습니다.
		   
		   백엔드 사용기술에 비해서 프론트엔드의 퀄리티가 떨어져서 잘 표현하지 못한것 저도 공감합니다.
		   현재는 css프레임워크로 원하는 퀄리티의 화면을 만들수있지만 당시에는 백엔드 개발하는것에 시간을 더 쓰는것이 낫다고 판단하여
		   그렇게 되었습니다.
	-> 화면 디자인에 대한 퀄리티가 낫다고 생각이 들지 않을정도로만 "깔끔"하게.
	톤앤매너(분위기, 색상은 3가지 색상 이하로 사용. 메인컬러지정해서 채도, 명도만 변경)에 통일감만 주고 심플하게.
	
	<프로젝트>

		(안좋은 역할분담)
		회원가입 화면 : A
		자바코드 : B
		쿼리, DB : C
		
		통일감을 주기위한 규칙, 공통작업을 초반에 시간들여서 해야함.
		그래도 통일감이 어긋난것은 제일 잘하는사람 또는 먼저 시간이 시간이 좀 남은사람, 그 분야로 취업하고싶은사람이 마무리.
2교시 : CDN
	Content Delevery Network
	물리적거리
	매우많은 요청을 하나의 서버에서 감당해야함
	(정적. 변하지않을)컨텐츠 전달이라는 용도에 특화된 서비스
	DNS에 CDN의 연락처가 있어야한다. 가장 빠른 EDGE를 연결해줌
	정적캐싱 : 미리 지점에 갖다놓기
	동적캐싱 : 요청을 받으면 있으면 쓰고, 없으면 가져오기
	장점1. 대역폭 대역을 줄여든다. 호스팅 비용절감.
	장점2. 가용성, 안정성 향상
	장점3. 보안향상. DDOS, 컨텐츠 암호화 등

	JQEURY : 이 브라우저에 되는거 저 브라우저에서 안된다. 그런것을 해결해주는 라이브러리.
		요즘은 브라우저들의 표준화가 잘 진행되어서 jqeury에 대한 의존성이 낮아지고 있지만
		여전히 대부분의 웹에 포함되어있다.
		1. jqeury 소스 다운받기
		2. 선택후 함수 호출
		css든 자바스크립트든   어떤것에 어떤 디자인을 줄지,    어떤것에 어떤 기능을 줄지
		선택자 { 본문 }
		JQEURY 문법 : $(선택자).함수();
		
		jquery에는 매우 많은 기능이 있는데 실질적으로 사용하는 기능은 몇개 안된다.
		그래서 사용하는 기능에 비해 용량이 너무 커서 요즘같은 브라우저의 표준화가 잘된 시점에는 점차 걷어내고 있는 실정.
		선택자를 쉽게 사용할수있는 목적으로 쓴다고 해도 과언이 아니다.
		
		자바스립트로 구현하려고 하는데 잘 안된다. 근데 검색해보니까 jqeury구현된건 있다. 그때 jquery쓰면된다.
		
	실습. lorem1클릭하면 alert("hello world") 띄워보기  //순수자바스크립트로. onclick쓰지말고
	//dom요소에서 lorem1을 선택하고     클릭이벤트리스너 달고,    클릭되면 실행할 본문 넣기.
